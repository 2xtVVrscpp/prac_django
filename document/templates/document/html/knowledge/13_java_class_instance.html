インスタンスとクラス¶

オブジェクトを作る上での注意¶

  1. 各オブジェクトの責務を考えた上で，フィールド，操作を定義する．

  2. 各クラスからオブジェクトを仮想世界に複製し，動く．

その物を作っているわけではなく，設計図を作っている．

クラスとオブジェクトを分ける意味¶

オブジェクトを直で使いたい人もいるかと思いますが， 例えばあるデータを3つ複製してくださいと言われた時， いちいち，1つずつ作るよりは 大元のファイルをコピペするかと思います． Javaもそれに倣ってコピペします．そのほうが効率が良いのです．

この時，設計図をクラス．そこからコピペされたものをインスタンスといいます．

プログラム中では， 設計図の書き換えは行わず，インスタンスの方を編集します．

例：クラスとして人間を定義
インスタンス生成で，できた人間にAという名前をつけ， 育つ過程で，性格や身長といった属性が代入される．
